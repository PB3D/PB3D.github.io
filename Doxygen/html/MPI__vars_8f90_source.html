16  ! lock type
17  ! There is a blocking (BL) and a nonblocking (NB) version where the former
18  ! requires an exclusive lock and the latter a shared one. This is saved in
19  ! the variable "blocking".
20  ! NB processes that get the lock directly on request (meaning that there
21  ! were no other processes in the queue) notify directly all the next NB
22  ! processes after gaining access. It also sets their status to active. When
23  ! a NB process gains the lock when notified after waiting, it does not have
24  ! to check for other NB processes, as this has been done by the notifying
25  ! process.
26  ! A BL process retains exclusive access upon receipt of the lock. Similarly
27  ! to NB processes, if the receipt was direct on request, the status is set
28  ! to active, but only of this NB process.
29  ! When returning the lock, all BL processes and NB that find themselves to
30  ! be the last active NB process, scan the waiting list and pass the lock
31  ! preferably to another BL process to notify. If not available, it searches
32  ! for all the NB processes to notify together.
33  ! The advantage of prefering BL processes after finishing a process is
34  ! that this way NB processes are accumulated, and then quickly finished
35  ! afterwards.
36  ! Note: every process in the waiting queue will eventually receive a
37  ! notification.
38  ! Scheme:
39  ! | request access | gain access | return access |
40  ! ---+---------------------+---------------------+--------------------+
41  ! | | | - remove from |
42  ! | | | queue |
43  ! BL | - add to queue | if direct: | - find next BL/NB |
44  ! | - wait | - activate | - notify |
45  ! | | | - activate all |
46  ! ---+---------------------+---------------------+--------------------+
47  ! | | | - remove from |
48  ! | | if direct: | queue |
49  ! NB | - add to queue | - find next NB | if last NB |
50  ! | - wait | next NB procs | - find next BL/NB |
51  ! | | - notify | - notify |
52  ! | | - activate all | - activate all |
53  ! ---+---------------------+---------------------+--------------------+
54  ! with preference BL > NB(s).
55  ! (extended from http://www.mcs.anl.gov/~thakur/papers/atomic-mode.pdf)

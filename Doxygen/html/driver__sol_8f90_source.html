262 !> \public Interpolate tensorial perturbation quantities in the third
263 !! dimension.
264 !!
265 !! The input grid should not be divided, though the output grid can be.
266 !!
267 !! The procedure considers all possible mode number combinations. For \c
268 !! X_style 2 (fast), each secondary mode only lives in a certain normal
269 !! range of the plasma. Therefore, each secondary mode pair also has a
270 !! limited normal range, given by the overlap of the ranges of the members.
271 !!
272 !! The interpolated mode number combinations have a normal range that might
273 !! slightly differ from the input ranges, in which case extrapolation can be
274 !! done if the method allows for it.
275 !!
276 !! \note If the input grid is too coarse, but the interpolated grid is not,
277 !! it is in theory possible that there are mode numbers and therefore mode
278 !! number pairs that exist in the input grid, though they do so in the
279 !! interpolated grid. In this case, they can not be calculated and
280 !! are currently set to zero. However, the current implementation of
281 !! setup_modes() produces an error if less than the full range of mode
282 !! numbers is accurately present.
283 integer function interp_v(mds_i,grid_i,X_i,mds_o,grid_o,X_o) result(ierr)

28  ! type for grids
29  ! The grids are saved in the following format: (angle_1,angle_2,r), where
30  ! angle_1 and angle_2 can be any angle that completely describe a flux
31  ! surface. For example, they can refer to a grid of theta and zeta values,
32  ! but they can also refer to (Modified) flux coordinates with a parallel
33  ! angle and a field line coordinate (alpha).
34  ! Normally, for field-aligned grids, angle_1 is chosen equal to the parallel
35  ! coordinate in the Flux coordinate system, and angle_2 equal to the field
36  ! line label (so the second dimension of the matrices is then chosen to be
37  ! of size 1 for the calculations on a single field line). At the same time,
38  ! the parallel coordinate, in which integrations will have to be done,
39  ! ocupies the first index. This is good for numerical efficiency.
40  ! For specific equilibrium grids, such as the case for HELENA equilibria,
41  ! angle_1 corresponds normally to theta and angle_2 to the symmetry angle
42  ! zeta.
43  ! It is important that this order of the coordinates in space is consistent
44  ! among all the variables.
45  type :: grid_type
46  integer :: n(3) ! tot nr. of points
47  integer :: loc_n_r ! local nr. of normal points
48  integer :: i_min, i_max ! min. and max. normal index of this process in full arrays
49  logical :: divided ! whether the grid is split up among the processes
50  real(dp), pointer :: r_e(:) => null() ! E(quilibrium) coord. values at n points
51  real(dp), pointer :: r_f(:) => null() ! F(lux) coord. values at n points

55  real(dp), pointer :: theta_f(:,:,:) => null() ! F(lux) coord. values of first angle at n points in 3D

82  ! Initializes a new grid.
83  ! Optionally, the local limits can be provided for a divided grid.
84  ! Optionally, it can be set whether the grid is divided or not. A situation
85  ! where this is useful is when only a subset of MPI processes is used to
86  ! calculate the solution in SLEPC. In this case, the extra ranks all only
87  ! contain the last grid point, and the last used process has as upper limit
88  ! this same grid point. This way, all the procedures are reusable, but in
89  ! this case if only one process is used, this procedure becomes confused and
90  ! sets this process to undivided. In most cases, this functionality probably
91  ! does not need to be used.
